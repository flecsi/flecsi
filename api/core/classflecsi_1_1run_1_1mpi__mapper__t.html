In case of FleCSI we want exact region that has been requested to be created. This is different from Default mapper which will map Parent region, if it exists.

Specialization of the map_task funtion for FleCSI.

The function has some FleCSI-specific features:

1) It specifies SOA ordering for new physical instances;

2) It stores information about already created instances and avoids creating a new instance if possible;

3) It has logic on how to create compacted instances;
